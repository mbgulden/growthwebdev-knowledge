## Current vs historical boundary

| Item | Treatment |
|---|---|
| Scan-triage branch outputs | Historical provenance. |
| Repeated generated files | Duplicate/superseded unless selected later with a clear current use. |
| Current Prismatic OKF structure | Use [canonical project index](../index.md), not raw Ned scan outputs. |

## What is safe to cite

- That Ned scan-triage ran and produced large OKF candidate families.
- That selected outputs are provenance for the OKF drift/recovery story.
- That unreviewed generated scan files are not current operational truth.

## What must not be treated as current operational truth

- Any dashboard/dispatcher/API claim inside a Ned scan file unless reverified against current `deploy-fresh` or current hub records.
- Any per-run generated TODO as an active obligation unless linked from current Linear/OKF.
