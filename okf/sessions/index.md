## What goes here

A session summary is filed when a single /yolo session or major work block produces durable artifacts. It includes:

- **TL;DR** — what shipped
- **Timeline** — what was built when
- **Linear issues** — what's filed, what's Done
- **Git commits** — the actual evidence
- **OKF docs** — the durable documentation
- **What's next** — the next concrete steps

## Why

Sessions produce a lot of intermediate state (PRs, commits, comments, intermediate docs). The summary captures the *durable* artifacts + the *decisions* + the *lessons learned* so future agents and humans can reconstruct what happened without reading every commit message.
